# OpenClaw Discord Bridge - Implementation Guide

## Overview

This document describes two approaches to enable Discord integration with full proxy support:

1. **Standalone Bridge** (RECOMMENDED) - Run independently from OpenClaw, forwards messages via HTTP API
2. **ChannelPlugin Implementation** - Integrate as a full OpenClaw channel plugin

## Approach 1: Standalone Bridge (RECOMMENDED)

The standalone bridge runs as a separate Node.js application that:
- Connects to Discord with full proxy support (HttpsProxyAgent)
- Forwards messages to OpenClaw's HTTP API
- Sends responses back to Discord via proxy

### Why This Approach?

- ✅ Works immediately without OpenClaw plugin integration complexity
- ✅ Full proxy support for WebSocket and REST API
- ✅ Independent lifecycle - can restart bridge without restarting OpenClaw
- ✅ Simpler debugging and testing
- ✅ No ChannelPlugin interface implementation required

### Files

- `/root/pluginupdate/bridge.ts` - Standalone bridge application
- `/root/pluginupdate/README.md` - Usage instructions

### Quick Start

```bash
cd /root/pluginupdate

# Run with ts-node
DISCORD_TOKEN="YOUR_BOT_TOKEN" \
PROXY_URL="http://127.0.0.1:7890" \
OPENCLAW_HTTP_URL="http://127.0.0.1:18789" \
OPENCLAW_AUTH_TOKEN="your-auth-token" \
npx ts-node bridge.ts
```

### Running as a Service

See `/root/pluginupdate/README.md` for systemd service configuration.

## Approach 2: ChannelPlugin Implementation

This approach integrates the discord-proxy as a full OpenClaw channel plugin.
See the detailed specification below.

## Problem Statement

The official OpenClaw Discord channel has a bug (issue #27409) where:
- Gateway WebSocket connections respect the proxy configuration ✓
- REST API calls (sending messages) do NOT respect the proxy configuration ✗
- Result: Bot can receive messages but cannot send replies, showing "fetch failed" errors

## Current State

### Plugin Location
- Installed: `/usr/lib/node_modules/openclaw/extensions/discord-proxy/`
- Source for modification: `/root/pluginupdate/`

### What Works
- Plugin loads and appears in plugin list
- Proxy configuration is accepted
- Discord bot can connect via proxy
- Messages are received from Discord

### What Doesn't Work
- Plugin cannot be enabled as a channel
- Config validation fails with "unknown channel id: discord-proxy"
- Missing ChannelPlugin interface implementation

### 2. Key Integration Points

#### A. Proxy Injection in outbound.sendText

This is the critical fix for issue #27409. The `sendText` method must use the proxy-aware Discord client:

```typescript
sendText: async ({ to, text, accountId, deps, replyToId, silent }) => {
  const account = resolveDiscordProxyAccount({ cfg, accountId });
  const proxyUrl = account.proxy;
  
  // Create HttpsProxyAgent for this request
  const { HttpsProxyAgent } = await import('https-proxy-agent');
  const proxyAgent = new HttpsProxyAgent(proxyUrl);
  
  // Use discord.js with proxy agent
  const { Client, GatewayIntentBits } = await import('discord.js');
  const client = new Client({
    intents: [GatewayIntentBits.Guilds, GatewayIntentBits.DirectMessages],
    rest: { agent: proxyAgent as any }, // KEY FIX: Inject proxy agent
  });
  
  await client.login(account.token);
  
  const channel = await client.channels.fetch(to);
  if (channel && 'send' in channel) {
    await channel.send({ content: text });
  }
  
  await client.destroy();
  
  return { channel: "discord-proxy", messageId: "..." };
}
```

#### B. Gateway Account Management

```typescript
gateway: {
  startAccount: async (ctx) => {
    const account = ctx.account;
    const token = account.token.trim();
    const proxyUrl = account.proxy || ctx.cfg.channels?.["discord-proxy"]?.proxy;
    
    // Import the existing proxy client creation logic
    const { createDiscordClientWithProxy } = await import('./dist/client.js');
    
    const client = createDiscordClientWithProxy(token, proxyUrl);
    
    // Set up message handlers
    client.on(Events.MessageCreate, async (message) => {
      // Forward to OpenClaw via ctx.send()
    });
    
    await client.login(token);
    
    // Return cleanup function
    return async () => {
      await client.destroy();
    };
  },
}
```

### 6. Configuration

After successful implementation:

```bash
# Disable official Discord channel
openclaw config set channels.discord.enabled false --json

# Enable discord-proxy channel
openclaw config set channels.discord-proxy.enabled true --json
openclaw config set channels.discord-proxy.token "YOUR_BOT_TOKEN" --json
openclaw config set channels.discord-proxy.proxy "http://127.0.0.1:7890" --json
openclaw config set channels.discord-proxy.groupPolicy "open" --json

# Restart gateway
openclaw gateway restart
```
